Getting Reacquainted With Git

June 22, 2020

Between making my first and second blog posts I forgot the basic git commands needed to update this website. While I managed to set the site up and understood the basics of a VCS (version control system) and git, I hadn’t learned the process in a way that really sunk in. We all learn differently and I tend to learn better by doing rather than purely reading documentation. So when reacquainting myself with git I thankfully found Jessica Lord’s quick tutorial app Git-it.

Git-it is not meant to be an all encompassing git tutorial. It is meant for people just dipping their toes in the pool. Over the course of the 11 lessons you set up git on your computer, open a GitHub account, create a repository, and learn the basics of committing, forking and merging. The lessons are helpful even if you don’t want to use GitHub for hosting.

Here are the basic commands to update my simple static website.

To check the website repository for any changes:

$ git status

To add any changed files for an update:

$ git add 'filename'

(To update all changed files you can use $ git add -a)

To commit the files being updated and add a message regarding the changes:

git commit -m “message”

The final step of sending the updated files to the GitHub repository:

$ git push origin master

Two caveats to the Git-it lessons. Firstly, a few of the Git-it lessons show outdated UI on the GitHub site and so you will have to do a little problem solving to accomplish some of the tasks. This isn’t a big deal as the basic concepts are explained; it’s just that the GitHub site has been updated. Secondly, if you have enabled two factor authentication (2FA) to your GitHub account you will need to set up a token in order to push any changes from your terminal. Git-it won't warn you about this issue. See the help pages for details.
